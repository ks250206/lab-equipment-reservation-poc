# 実装内容（イテレーション 19 — 装置写真・MinIO・一覧表示モード）

## スコープ

- `devices` に画像オブジェクトキー・Content-Type を保持し、MinIO（S3 互換）に PNG/JPEG を保管する。
- 管理者向け `POST /api/devices/{id}/image`（容量上限・マジックバイトで PNG/JPEG のみ許可）。
- 公開 `GET /api/devices/{id}/image` で画像バイト列を返す（無いときは 404。フロントでプレースホルダ）。
- 開発 compose に MinIO、`.env` でエンドポイント・鍵・バケット名を設定。
- シードで各装置に 1x1 PNG を投入。
- 装置詳細はメイン＋右カラムに画像（またはプレースホルダ）。管理者はアップロード UI。
- 装置一覧にリスト／詳細／サムネの表示切替（**既定はサムネ**）。

## 非目標

- 画像の CDN 署名 URLや複数枚ギャラリー。
- 本番用の IAM 最小権限設計の詳細。

## 補足

- `boto3` で S3 API を使用。非同期ルート内は `asyncio.to_thread` で同期クライアントを呼ぶ。
