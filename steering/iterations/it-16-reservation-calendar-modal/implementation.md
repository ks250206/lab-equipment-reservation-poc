# 実装内容（イテレーション 16 — リスト上ページネーション・カレンダー表示・予約モーダル）

## スコープ

- 装置一覧・装置予約リストで `ListPaginationBar` をリスト／表の上に配置。
- カレンダーイベントの表示を「開始–終了（ローカル）＋氏名（なければメール等）」に統一し、`extendedProps` に予約本体と `isMine` を載せる。
- 月ビューのみ `dayMaxEvents` と「+N件」形式の more リンク。
- イベントクリックで詳細モーダル（Radix Dialog）。他人は閲覧のみ、自分は開始・終了・目的・ステータスを編集して `PUT /api/reservations/{id}` で保存。
- `datetime-local` ↔ ISO の共通化（`ReservationsPage` から参照）。

## 非目標

- カレンダー上でのドラッグによる日時変更。
- 他人の予約の管理者編集（別 API の追加）。

## 補足

- 予約更新 API は既存の所有者チェックのみ利用する。
